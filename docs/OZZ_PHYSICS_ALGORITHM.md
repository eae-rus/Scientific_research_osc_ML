# Физический (детерминированный) алгоритм детектирования ОЗЗ и ДПОЗЗ

## 1. Назначение

Алгоритм реализует классические электротехнические критерии для классификации
однофазных замыканий на землю (ОЗЗ) в сетях 6–35 кВ с изолированной или
компенсированной нейтралью.

Используется как **детерминированный baseline** для сравнения с нейросетевыми
моделями (cPhysicsKAN и др.) в рамках эксперимента 2.6.11.

**Файл реализации:** `osc_tools/analysis/ozz_physics.py`

---

## 2. Целевые классы

| Класс | Колонка разметки | Описание |
|-------|------------------|----------|
| **Target_OZZ** | ML_2_1 ∨ ML_2_1_1 ∨ ML_2_1_2 ∨ ML_2_1_3 | Любое ОЗЗ (обобщение) |
| **Target_OZZ_decay** | ML_2_1_2 | Затухающее ОЗЗ (подмножество OZZ) |
| **Target_OZZ_dpozz** | ML_2_1_3 | ДПОЗЗ — дуговые перемежающиеся замыкания (подмножество OZZ) |

> **Важно:** Классы НЕ взаимоисключающие. Если `Target_OZZ_dpozz = 1`, то обязательно `Target_OZZ = 1`.
> Это multi-label задача с иерархической структурой.

## 3. Входные данные

### 3.1. Формат окна

Алгоритм принимает **одно окно** данных — массив формы `(T, 8)`:

| Индекс | Канал | Описание |
|--------|-------|----------|
| 0 | IA | Ток фазы A |
| 1 | IB | Ток фазы B |
| 2 | IC | Ток фазы C |
| 3 | IN | Ток нулевой последовательности |
| 4 | **UA** | **Напряжение фазы A** |
| 5 | **UB** | **Напряжение фазы B** |
| 6 | **UC** | **Напряжение фазы C** |
| 7 | UN | Напряжение нулевой последовательности |

Алгоритм **использует только каналы напряжений** (UA, UB, UC) — индексы 4, 5, 6.

### 3.2. Нормализация

Все напряжения в датасете **предварительно нормализованы**:

$$
U_{\text{norm}} = \frac{U_{\text{raw}}}{3 \cdot U_{b,\text{base}}}
$$

где $U_{b,\text{base}}$ — базовый коэффициент напряжения (типично 100 В для фидеров 6 кВ или 400 В для 35 кВ).

Статистики нормализованных данных:
- $\mu(U) \approx 0$, $\sigma(U) \approx 0.146$
- Диапазон: $[-0.44, +0.44]$ (в нормальном режиме)
- Амплитуда фазного напряжения: ~0.3–0.5 (в о.е.)

### 3.3. Параметры окна

| Параметр | Значение | Описание |
|----------|----------|----------|
| `window_size` | 320 | Число отсчётов в окне |
| `fs` | 1600 Гц | Частота дискретизации |
| Длительность | 200 мс | = 10 периодов промышленной частоты (50 Гц) |

## 4. Алгоритм (пошагово)

### 4.1. Шаг 1: Вычисление напряжения нулевой последовательности 3U₀

$$
3U_0(t) = U_A(t) + U_B(t) + U_C(t)
$$

В **нормальном** (симметричном) режиме:

$$
3U_0 \approx 0
$$

так как фазные напряжения сдвинуты на 120°, и их сумма компенсируется.

При **ОЗЗ** симметрия нарушается, и $3U_0$ приобретает значительную амплитуду (порядка фазного напряжения).

### 4.2. Шаг 2: Скользящий спектральный анализ 3U₀

Действующее значение первой гармоники и коэффициент гармоник (THD) вычисляются **поточечно** — 
скользящим окном в 1 период сети ($N_T = f_s / f_0 = 32$ отсчёта при 1600 Гц / 50 Гц).

Для каждой позиции $i$ (от 0 до $T - N_T$):

1. Вычисляется БПФ (RFFT) окна $3U_0[i : i+N_T]$.
2. Извлекается первая гармоника (индекс 1):
   $$\text{RMS}_{f_0}[i] = \frac{|X_i(1)| \cdot 2}{N_T \cdot \sqrt{2}}$$
3. Вычисляется суммарное RMS высших гармоник (индексы от 2 до $N_T/2$):
   $$\text{RMS}_{\text{harm}}[i] = \sqrt{\sum_{k=2}^{N_T/2} \left(\frac{|X_i(k)| \cdot 2}{N_T \cdot \sqrt{2}}\right)^2}$$
4. Вычисляется мгновенный коэффициент гармоник THD:
   $$\text{THD}[i] = \frac{\text{RMS}_{\text{harm}}[i]}{\text{RMS}_{f_0}[i] + 10^{-9}}$$

Результат — массивы `u0_rms_arr` и `thd_arr` длины $(T - N_T + 1)$.

**Критерий базового ОЗЗ (Класс 0):**

$$
\max(\text{RMS}_{f_0}[\cdot]) < u_0^{\text{threshold}} \implies \text{Нормальный режим (None)}
$$

Реализация: `_rms_fundamental_sliding()` (векторизована через `sliding_window_view` + batch `rfft`).

### 4.3. Шаг 3: Производная 3U₀ (для дополнительной детекции пробоев)

$$
\frac{d(3U_0)}{dt} \approx \frac{3U_0[n+1] - 3U_0[n]}{\Delta t}, \quad \Delta t = \frac{1}{f_s}
$$

Вычисляется стандартное отклонение производной $\sigma_{dU_0}$ для поиска аномальных пиков (пробоев изоляции).

### 4.4. Шаг 4: Критерий ДПОЗЗ (Класс 2) - Спектральный метод

ДПОЗЗ (дуговые перемежающиеся замыкания) генерируют богатый спектр высших гармоник из-за несинусоидальной формы напряжения (ступеньки, перезаряды).

**Алгоритм:**

1. Определяется «активная область» окна, где $\text{RMS}_{f_0} \geq u_0^{\text{threshold}}$.
2. На этой области вычисляется 90-й перцентиль THD.
3. **Решение ДПОЗЗ по THD:**
   $$P_{90}(\text{THD}_{\text{active}}) > \text{thd\_threshold} \implies \text{ДПОЗЗ (2)}$$

4. **Резервный критерий (пики производной):**
   Если по THD класс не определен, проверяется наличие резких фронтов:
   $$N_{\text{peaks}}(|dU_0/dt| > 5\sigma) \geq N_{\text{min}}^{\text{dpozz}} \implies \text{ДПОЗЗ (2)}$$

### 4.5. Шаг 5: Критерий затухающего ОЗЗ (Класс 1) - Анализ тренда RMS

Затухающее ОЗЗ характеризуется плавным спадом амплитуды $3U_0$ после первоначального пробоя.

**Алгоритм:**

1. Используется массив скользящего RMS первой гармоники `u0_rms_arr`.
2. Находится позиция максимума: $i_{\max} = \arg\max(\text{RMS}_{f_0})$.
3. Проверяется, что пик в первой части окна ($i_{\max} < 0.7 \cdot \text{Window}$), чтобы было время для затухания.
4. Вычисляется «хвост» RMS (среднее последних 15% окна).
5. **Решение о затухании:**
   $$\frac{\text{RMS}_{\text{tail}}}{\text{RMS}_{\max}} < \text{decay\_ratio} \implies \text{Затухающее ОЗЗ (1)}$$

## 5. Уставки (параметры алгоритма)

| Параметр | По умолчанию | Описание | Обоснование |
|----------|-------------|----------|-------------|
| `u0_threshold` | **0.0333** | Порог RMS 3U₀ (в о.е.) | Соответствует 10В вторичных. Отсекает фоновую несимметрию. |
| `thd_threshold` | **0.15** | Порог коэффициента гармоник | 15% THD — типичный маркер несинусоидальности при горении дуги. |
| `decay_ratio` | **0.30** | Порог затухания RMS | Если амплитуда в конце окна упала ниже 30% от пика — признак затухания. |
| `deriv_peak_sigma` | **5.0** | Кратность σ для dU₀/dt | Порог для выделения резких фронтов перезаряда емкостей. |
| `min_dpozz_peaks` | **3** | Минимум пиков для ДПОЗЗ | Требуется минимум 3 пробоя за 200мс для резервного критерия. |
| `rms_tail_fraction` | **0.15** | Доля окна для оценки затухания | Последние 15% (30 мс) используются для расчета среднего "хвоста". |
| `operate_delay_periods` | **0.0** | Выдержка срабатывания | Блокировка, если RMS выше порога держится меньше заданного времени. |


### 5.1. Пересчёт порога для ненормализованных данных

Если используются **сырые** (ненормализованные) данные:

$$
u_0^{\text{threshold,raw}} = u_0^{\text{threshold,norm}} \times 3 \times U_{b,\text{base}}
$$

Примеры:
- $U_{b,\text{base}} = 100$ В → $u_0^{\text{norm}} = 10 / 300 = 0.0333$
- $U_{b,\text{base}} = 400$ В → $u_0^{\text{norm}} = 10 / 1200 = 0.0083$

## 6. Блок-схема алгоритма

```
┌─────────────────────────────────────────────┐
│            Входное окно (T, 8)              │
└──────────────────┬──────────────────────────┘
                   ▼
┌──────────────────────────────────────────────┐
│  3U₀(t) = UA(t) + UB(t) + UC(t)              │
└──────────────────┬───────────────────────────┘
                   ▼
┌──────────────────────────────────────────────┐
│  Скользящее RMS (окно = 1 период = 32 отсч.) │
│  u0_rms_arr = sliding_rfft(3U₀, N_period)    │
│  max(u0_rms_arr) < u0_threshold ?             │
│  ДА → return None (нормальный режим)          │
└──────────────┬───────────────────────────────┘
               ▼ НЕТ
┌──────────────────────────────────────────────┐
│  Выдержка времени: delay_samples ПОДРЯД      │
│  идущих отсчётов RMS ≥ порога?               │
│  НЕТ → return None                           │
└──────────────┬───────────────────────────────┘
               ▼ ДА (базовый ОЗЗ подтверждён → active = {0})
┌──────────────────────────────────────────────┐
│  dU₀/dt = diff(3U₀) / Δt                     │
│  Поиск пиков: |dU₀/dt| > 5σ                  │
│  THD = RMS_harm / RMS_fund (спектр 1 пер.)   │
│  RMS Trend: idx_max, mean(tail)              │
└──────────────┬───────────────────────────────┘
               ▼
┌──────────────────────────────────────────────┐
│  P90(THD) > 0.15 ИЛИ N_peaks ≥ 3?            │
│  ДА → active.add(2)  (ДПОЗЗ)                │
└──────────────┬───────────────────────────────┘
               ▼
┌──────────────────────────────────────────────┐
│  RMS_tail < 0.3·RMS_max И peak_pos < 0.7·T?  │
│  ДА → active.add(1)  (Затухающее ОЗЗ)       │
└──────────────┬───────────────────────────────┘
               ▼
┌──────────────────────────────────────────────┐
│  return active  (set: multi-label результат) │
└──────────────────────────────────────────────┘
```

## 7. Формирование multi-label вектора

Алгоритм классификации возвращает **набор** активных классов (`Set[int]`),
т.к. ДПОЗЗ и затухающее ОЗЗ могут наблюдаться одновременно в одном окне.
Класс 0 (базовый ОЗЗ) — обязательное условие для любых подтипов.

- `{0}` = устойчивое ОЗЗ, `{0, 1}` = затухающее, `{0, 2}` = ДПОЗЗ,
  `{0, 1, 2}` = ДПОЗЗ + затухающее, `None` = норма.

При формировании вектора `(Target_OZZ, Target_OZZ_decay, Target_OZZ_dpozz)`:

| Результат алгоритма   | Target_OZZ | Target_OZZ_decay | Target_OZZ_dpozz |
|-----------------------|-----------|------------------|------------------|
| None                  | 0         | 0                | 0                |
| {0} (устойчивое)      | **1**     | 0                | 0                |
| {0, 1} (затухающее)   | **1**     | **1**            | 0                |
| {0, 2} (ДПОЗЗ)        | **1**     | 0                | **1**            |
| {0, 1, 2} (оба подтипа)| **1**   | **1**            | **1**            |

Выдержка времени: алгоритм требует `delay_samples` **подряд** идущих отсчётов
с RMS выше порога (используется `_max_consecutive_run`). Это предотвращает
ложные срабатывания на кратковременных всплесках.

Этот пересчёт выполняется в `evaluate_physics_baseline.py`.

## 9. Ограничения и допущения

1. **Только напряжения.** Алгоритм не использует токи. Это ограничивает точность при слабых ОЗЗ с малым 3U₀.

2. **Поточечное RMS, оконная агрегация.** RMS первой гармоники вычисляется
   скользящим окном в 1 период (32 отсчёта). Классификация ДПОЗЗ/затухающего
   ОЗЗ выполняется по совокупности признаков внутри «большого» окна (320 отсчётов).
   Поточечная классификация базового ОЗЗ возможна напрямую по массиву `u0_rms_arr`.

3. **Фиксированная частота.** Оптимизирован для $f_0 = 50$ Гц и $f_s = 1600$ Гц.

4. **Нормализация.** Пороги настроены для нормализованных данных. При изменении схемы нормализации необходимо пересчитать `u0_threshold`.

5. **Граничные случаи.** В реальных осциллограммах ДПОЗЗ с малым числом пробоев (1–2 за 200 мс) может быть ошибочно классифицирован как устойчивое ОЗЗ.
