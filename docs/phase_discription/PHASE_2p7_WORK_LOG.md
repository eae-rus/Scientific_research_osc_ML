## 11 мая 2026 — Инициализация Фазы 2.7

### Выполнено:

1. **5-кратная перекрёстная проверка (K-Fold CV)**
   - Реализован метод `DatasetManager.generate_cv_splits()` — стратифицированное разбиение
     по осциллограммам с учётом комбинаций меток (ML_1, ML_2, ML_3)
   - Сохранение в `data/ml_datasets/cv_splits.json` для воспроизводимости
   - Баланс: 198 тестовых / 792 обучающих файлов в каждом fold-е
   - Классы равномерно распределены: ML1≈27-28, ML2≈10-11, ML3≈20-21 на fold
   - Добавлены `load_cv_splits()` и `load_fold_data()` для удобной загрузки

2. **Новый скрипт `run_phase2_7_all.py`**
   - Полная переработка скрипта экспериментов с 5-ю вложенными циклами
   - Поддержка CLI для параллельного запуска на сервере
   - Поддержка ручного запуска через IDE (константы)
   - Автоматическая фильтрация несовместимых комбинаций модель/данные
   - Разделение на контексты статей (paper1: модели, paper2: данные)

3. **Множественные seeds** 
   - 5 seeds с простыми числами в качестве смещений: [0, 7919, 15731, 23497, 31607]
   - Обеспечивают максимальное различие инициализаций

4. **Поддержка rPhysicsKAN в runner**
   - Добавлен импорт и инстанцирование `rPhysicsKAN` в `ExperimentRunner`

5. **Тестовый прогон**
   - Успешно обучена модель `SimpleMLP_light_raw_stride` на Fold 1, Seed 0, 1 эпоха
   - Подтверждена корректность: загрузка данных, создание индексов, обучение, валидация,
     сохранение чекпоинтов

### Технические решения:

- **Валидационная выборка**: решено НЕ выделять отдельную (train/val/test),
  т.к. при 51 файле ML_2 и 5-fold CV это даёт ~10 файлов ML_2 в тесте — 
  дальнейшее дробление нецелесообразно.
- **Аугментация**: всегда включена (стандартный набор из Фазы 2.5).
- **Взвешивание**: BCEWithLogitsLoss с pos_weight по частоте событий.
- **Обратная совместимость**: все изменения аддитивны, старые скрипты не затронуты.
