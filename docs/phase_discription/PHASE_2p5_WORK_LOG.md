# Лог работы по Фазе 2.5 (Data Engineering & Preprocessing)

Этот файл содержит хронологию изменений и технических решений, принятых в ходе выполнения Фазы 2.5. Ведется в обратном хронологическом порядке.

## 2026-01-06 (Part 9): Feature Engineering & Pipeline Refinement

### Основные изменения
1. **Расширение `OscillogramDataset` для поддержки гармоник**:
   - Обновлен метод `__init__` для приема параметра `num_harmonics`.
   - В режимах `symmetric` и `symmetric_polar` реализован расчет нескольких гармоник (с помощью `sliding_window_fft`).
   - Для `symmetric_polar` каждая гармоника преобразуется в Magnitude/Angle (относительно фундаментальной Reference).
   - Размерность входных данных теперь динамически зависит от количества гармоник: `12 * num_harmonics` каналов.

2. **Обновление раннера `run_phase2_5_all.py`**:
   - Реализована логика выбора параметров (stride, harmonics) в зависимости от запрашиваемой сложности (`complexity`) и стратегии сэмплирования:
     - **Light**: 1 гармоника, шаг прореживания 2 (для плотных данных).
     - **Medium**: 3 гармоники, шаг прореживания 32.
     - **Heavy**: 9 гармоник (максимальная информативность для Snapshot).
   - Добавлены подробные инструкции по запуску и параметрам в `if __name__ == "__main__":`.

3. **Корректировка планов экспериментов**:
   - Уточнены параметры запуска для стратегий (соответствие сложности и количества гармоник).

## 2026-01-06 (Part 8): Model Scaling & Tuning (Stage 4.5)

## 2026-01-05 (Part 5): Class Balancing & Arithmetic Layers

### Основные изменения

        -   `MultiplicationLayer`: Поэлементное умножение (для моделирования мощности $S=UI$).

    -   Добавлены unit-тесты `tests/unit/test_kan_arithmetic.py`.

## 2026-01-05 (Part 4): Input Strategies & Multi-Label

        -   `snapshot`: Взятие только первой и последней точки окна.

## 2026-01-05 (Part 3): Data Augmentation

### Основные изменения
1.  **Модуль аугментации (`osc_tools/ml/augmentation.py`)**:
    -   Реализован класс `TimeSeriesAugmenter` для применения физически обоснованных преобразований к тензорам.
    -   Поддерживаемые методы:
        -   **Inversion**: Инверсия сигнала (умножение на -1).
        -   **Scaling**: Масштабирование амплитуды (отдельно для токов и напряжений).
        -   **Gaussian Noise**: Добавление шума.
        -   **Offset**: Смещение (DC offset).
        -   **Phase Shuffling**: Циклическая перестановка фаз (A->B->C->A).
        -   **Drop Channel**: Зануление случайного канала.
    -   Работает как с NumPy массивами, так и с PyTorch тензорами.

2.  **Интеграция в `OscillogramDataset`**:
    -   В `__init__` добавлен параметр `augmentation_config`.
    -   В `__getitem__` аугментация применяется к стандартизированным сырым данным (8 каналов) *перед* расчетом вторичных признаков (симметричные составляющие, мощность).
    -   Это гарантирует, что все производные признаки (например, I1, I2, P, Q) будут корректно отражать изменения в сырых данных.

3.  **Тестирование**:
    -   Создан файл тестов `tests/unit/test_augmentation.py`.
    -   Проверена корректность работы всех методов аугментации.
    -   Проверена интеграция с `OscillogramDataset` (включая проверку влияния масштабирования на симметричные составляющие).
    -   Выявлен и учтен нюанс с Hanning window в `sliding_window_fft` (ослабление амплитуды в 2 раза).

## 2026-01-05 (Part 2): Global Normalization & Sliding Window

### Основные изменения
1.  **Глобальная нормализация (Global Normalization)**:
    -   Реализован скрипт `scripts/calculate_dataset_stats.py` для расчета глобальных статистик (Mean, Std) по всему датасету.
    -   Скрипт обрабатывает 990 файлов (~1.5 млн строк) за ~2 минуты.
    -   **Результаты**:
        -   Токи: Mean ~0.001, Std ~1.59
        -   Напряжения: Mean ~0.01, Std ~53.4
    -   Исправлена обработка типов данных в `dataset.py`: добавлено явное приведение к `Float32` и `try-except` блоки для защиты от некорректных данных (строк, None) при чтении Polars DataFrame.

2.  **Sliding Window & Random Sampling**:
    -   В `OscillogramDataset` внедрена поддержка двух стратегий сэмплирования:
        -   **Train Mode (Random Sampling)**: Вместо фиксированного начала, теперь выбирается случайное окно внутри файла. Индексы передаются как кортежи `(start, length)`.
        -   **Val/Test Mode (Sliding Window)**: Реализована генерация фиксированных окон с заданным шагом (stride) для полного покрытия осциллограммы.
    -   Добавлен статический метод `OscillogramDataset.create_indices` для генерации списка индексов в зависимости от режима.
    -   Реализованы unit-тесты `tests/unit/test_dataset_sliding_window.py`, покрывающие обе стратегии.

3.  **Технические улучшения**:
    -   Исправлен `DeprecationWarning` в Polars (`with_row_count` -> `with_row_index`).
    -   Оптимизирована сортировка индексов при группировке файлов для детерминированного поведения.

## 2026-01-05: Интеграция нейтральных каналов (In/Un) и валидация данных

### Основные изменения
1.  **Расширение модели данных до 4-х проводной системы**:
    -   В `OscillogramDataset` добавлена полноценная поддержка каналов нулевой последовательности: тока (`In`) и напряжения (`Un`).
    -   Стандартизированный выход "сырых" данных (`raw`) теперь содержит **8 каналов**: `[IA, IB, IC, In, UA, UB, UC, Un]`.

2.  **Умный селектор каналов (Smart Selector) v2**:
    -   **Валидация сигналов**: Добавлена проверка на "значимость" данных. Сигналы с амплитудой ниже порога (0.01 А для тока, 0.1 В для напряжения) считаются отсутствующими/шумом.
    -   **Токи**: Метод `_get_best_current_channels` теперь возвращает 4 канала. Если `In` отсутствует в исходных данных, он заменяется массивом нулей.
    -   **Напряжения**: Метод `_get_best_voltage_channels` возвращает 4 канала. Реализована логика проброса `Un` при его наличии, даже если основные напряжения восстанавливаются из линейных.

3.  **Обновление Feature Engineering**:
    -   **Symmetric Mode**:
        -   Расчет `I0` и `U0` теперь использует прямые измерения `In` и `Un` (как `Xn / 3`), если они доступны.
        -   Размерность выхода: 12 каналов (6 для токов + 6 для напряжений: Re/Im для 1, 2, 0 последовательностей).
    -   **Power Mode**:
        -   Расчет активной (P) и реактивной (Q) мощности теперь производится для всех 4-х фаз (A, B, C, N).
        -   Размерность выхода увеличена с 6 до **8 каналов**.
    -   **Complex Channels**:
        -   Возвращает комплексные спектры для всех 8 каналов.
        -   Размерность выхода увеличена с 12 до **16 каналов**.

4.  **Рефакторинг и оптимизация**:
    -   Математическое ядро (расчет мощностей, симметричных составляющих, линейных напряжений) вынесено в отдельный модуль `osc_tools/features/phasor.py`.
    -   Реализован расчет симметричных составляющих напрямую из линейных напряжений (функция `calculate_symmetrical_components_from_line`), что повышает точность при отсутствии фазных замеров.

5.  **Тестирование**:
    -   Обновлены юнит-тесты `tests/unit/test_dataset_smart_selector.py` и `tests/unit/test_ml_dataset.py`.
    -   Скорректированы проверки размерностей (Shapes) тензоров.
    -   Добавлены тестовые сценарии для проверки валидации по epsilon и обработки отсутствующих каналов.
