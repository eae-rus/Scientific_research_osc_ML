# Отчет по Фазе 2: Исследование архитектур и оптимизация KAN для классификации аварийных режимов

## 1. Введение

Вторая фаза исследований была посвящена переходу от предварительных экспериментов к полноценному обучению и сравнительному анализу моделей на реальных данных. Основной фокус был сделан на исследовании применимости сетей Колмогорова-Арнольда (KAN) для задачи классификации аварийных режимов в энергосистемах, а также их сравнению с классическими архитектурами (MLP, CNN, ResNet).

Ключевой особенностью фазы стал **Data-Centric подход**: мы не просто подавали сырые данные в модели, а формировали физически обоснованные признаки (симметричные составляющие, мощности), что позволило значительно улучшить качество классификации.

---

## 2. Данные и их подготовка

### 2.1. Входные данные и формирование выборки
В качестве исходных данных использовались осциллограммы в формате COMTRADE.
- **Объем данных:** ~990 уникальных осциллограмм.
- **Размер окна:** 640 точек (при частоте дискретизации 1600 Гц это соответствует 0.4 секунды или 20 периодам промышленной частоты 50 Гц).

**Стратегия выборки (Sampling Strategy):**
В текущей реализации Фазы 2 использовался упрощенный подход к формированию выборки:
*   Из каждой осциллограммы (которая может содержать 4000+ отсчетов) бралось **только одно окно** — первые 640 отсчетов (начиная с индекса 0).
*   Это означает, что если аварийное событие начиналось позже 0.4 секунды, оно могло не попасть в обучающую выборку в полной мере. Это известное ограничение текущего этапа, которое будет устранено в Фазе 3 (см. раздел 5).

**Разделение данных (Train/Val Split):**
Разделение на обучающую и валидационную выборки производилось **пофайлово** (Group Split):
*   Список уникальных файлов перемешивался случайным образом.
*   80% файлов целиком уходили в Train, 20% — в Validation.
*   Это критически важно: фрагменты одной и той же осциллограммы никогда не попадают в разные выборки, что исключает "утечку данных" (data leakage) и завышение метрик.

### 2.2. Формирование признаков (Feature Engineering) и Нормализация

**Нормализация:**
В проекте реализован модуль `osc_tools/features/normalization.py`, который рассчитывает коэффициенты нормализации на основе номинальных параметров сети (110 кВ, 10 кВ и т.д.). Входные данные приводятся к относительным единицам (о.е.) путем деления на соответствующие базисные величины. Это обеспечивает стабильность обучения нейросетей, приводя значения токов и напряжений к сопоставимым диапазонам (обычно около 1.0 для номинальных режимов).

**Стратегии признаков:**

1.  **Raw (Сырые данные):**
    *   На вход подавались значения токов и напряжений как есть (после нормализации).
    *   Каналы: `IA, IB, IC, UA, UB, UC` (и другие доступные).

2.  **Symmetric Components (Симметричные составляющие) - *Основной метод*:**
    *   Используется скользящее окно FFT размером **32 отсчета** (1 период промышленной частоты при 1600 Гц / 50).
    *   Шаг скользящего окна: **1 отсчет**.
    *   Для каждого положения окна рассчитываются фазоры, а из них — симметричные составляющие ($I_1, I_2, I_0$ и $U_1, U_2, U_0$).
    *   **Размерность входа:** Из исходного окна в 640 точек получается последовательность длиной $640 - 32 + 1 = 609$ точек.
    *   Для каждой точки имеем 12 каналов (Re/Im части для 3 токов и 3 напряжений).
    *   *Комментарий:* Получение 609 точек из 640 является избыточным. В разделе "Планы" предложено использование шага (stride) 16 или 32 для уменьшения размерности.

### 2.3. Целевые метки (Выходные данные)
Задача решается как **Многоклассовая классификация (Multi-class classification)**.
*   **Формирование классов:** Метки формируются как уникальные комбинации активных флагов. Например, событие, где активны флаги "Фаза А" и "Земля", получает отдельный класс "A_Earth".
*   **Обработка пересечений:** Хотя физически классы могут пересекаться (однофазное КЗ является подмножеством аварийных режимов), в данной реализации каждая уникальная комбинация считается отдельным, взаимоисключающим классом для функции потерь `CrossEntropyLoss`.
*Комментарий:* В разделе "Планы" указал, что стоит начать рассматривать их как независимые и возможные к возникновению по отдельности, по идеии это будет многоголовость на выходе, то есть надо будет переработать архитектуры моделей.

## 3. Исследуемые модели

Выбор моделей обусловлен принципом "от простого к сложному" и необходимостью иметь надежный Baseline (точку отсчета).

### 3.1. SimpleMLP (Базовая полносвязная сеть)
*   **Вход:** Выпрямленный вектор (Flattened) всех отсчетов (например, $12 \times 609 = 7308$ значений).
*   **Архитектура:**
    *   Слой 1: Linear -> BatchNorm -> ReLU -> Dropout
    *   Скрытые слои: [128, 64] нейронов.
    *   Выход: Linear (число классов).
*   **Особенность:** Теряет временную структуру сигнала, смешивая все точки воедино. Служит простейшим бейзлайном.

### 3.2. SimpleCNN (Сверточная сеть)
*   **Вход:** Временной ряд (Channels x Time).
*   **Архитектура:**
    1.  **Block 1:** Conv1d(in, 16) -> BN -> ReLU -> MaxPool(2)
    2.  **Block 2:** Conv1d(16, 32) -> BN -> ReLU -> MaxPool(2)
    3.  **Block 3:** Conv1d(32, 64) -> BN -> ReLU -> AdaptiveAvgPool(1)
    4.  **Classifier:** Linear(64, 32) -> ReLU -> Linear(32, num_classes)
*   **Особенность:** Классическая архитектура, эффективно извлекающая локальные паттерны (фронты, пики) и инвариантная к небольшим временным сдвигам.

### 3.3. ConvKAN (Сверточная KAN)
*   **Вход:** Временной ряд.
*   **Архитектура:**
    *   Аналогична CNN, но вместо обычных сверток используются `KANConv1d`.
    *   Вместо линейного классификатора используются слои `KANLinear`.
*   **Принцип KAN:** Обучаемые функции активации (сплайны) находятся на ребрах графа вычислений. Это позволяет моделировать сложные нелинейные зависимости с меньшим числом параметров, но требует больше вычислительных ресурсов.

## 4. Результаты и Интерпретируемость

### 4.1. Результаты обучения
*   Базовые модели (MLP, CNN) быстро достигают точности ~78-87% (Accuracy).
*   Наблюдается проблема дисбаланса классов: модели склонны чаще предсказывать мажоритарный класс ("Normal"), что видно по метрикам (Accuracy высокая, но Loss может стагнировать).

### 4.2. Интерпретируемость KAN
*   **Прунинг (Pruning):** Реализован механизм прореживания KAN-сетей. Путем анализа L1-нормы функций активации удаляются слабые связи, что делает структуру сети более разреженной и понятной.
*   **Ограничения:** Для входных данных высокой размерности (640 отсчетов) полная визуализация графа KAN становится перегруженной и сложной для восприятия. Интерпретируемость KAN наиболее эффективна, когда на вход подаются уже высокоуровневые признаки (например, амплитуды гармоник, а не сырой сигнал), или когда сеть сильно прорежена.

### 4.3. Прореживание (Pruning) KAN
Мы провели исследование по удалению "слабых" связей в KAN (Pruning).
*   **Метод:** Удалялись связи, где L1-норма функции активации была меньше порога.
*   **Результат:** Удалось удалить **до 53%** связей без потери точности классификации.
*   **Значение:** Это доказывает, что KAN находит разреженную структуру задачи. В будущем это позволит создавать очень компактные и быстрые модели.

### 4.4. Интерпретируемость
Был создан инструмент для визуализации обученных сплайнов. Мы увидели, что KAN выучивает нетривиальные функции (не просто прямые линии или параболы), адаптируясь под данные. Карта внимания (Feature Importance) позволяет увидеть, на какие каналы сеть смотрит больше всего.

## 5. Планы по улучшению (Future Work)

На основе анализа результатов Фазы 2, предлагаются следующие улучшения для Фазы 3:

### 5.1. Улучшение работы с данными
*   **Скользящее окно (Sliding Window):** Вместо взятия только первых 640 отсчетов, необходимо нарезать длинные осциллограммы на множество перекрывающихся окон (например, с шагом 100 отсчетов). Это кратно увеличит обучающую выборку и позволит модели видеть развитие аварии во времени.
*   **Оптимизация входа (Downsampling):**
    *   Для симметричных составляющих использовать шаг (stride) **16 или 32** при расчете FFT. Это уменьшит длину входной последовательности с ~600 до ~20-40 точек без потери информативности (так как фазоры меняются медленно). Это существенно ускорит обучение и упростит модель.
    *   Рассмотреть возможность подачи на вход только выделенных событий (триггер по пусковому органу).

### 5.2. Архитектурные изменения
*   **Multi-label Classification:** Переход от `CrossEntropyLoss` к `BCEWithLogitsLoss` для независимого предсказания каждого типа повреждения. Это позволит корректно обрабатывать сложные аварии (например, "Земля" + "Фаза А" как два независимых флага).
*   **Hybrid Models:** Использование CNN для извлечения признаков из сырого сигнала и KAN для финальной классификации на основе этих признаков (где интерпретируемость наиболее важна).

### 5.3. Борьба с дисбалансом
*   Использование взвешенных функций потерь (Weighted Loss) для штрафа за ошибки на редких классах.
*   Оверсемплинг (Oversampling) редких аварийных классов при формировании батчей.
*   Val Loss (Raw): 1.1201
Это подтверждает гипотезу, что внедрение физических знаний (Physics-Informed ML) упрощает задачу для нейросети.
