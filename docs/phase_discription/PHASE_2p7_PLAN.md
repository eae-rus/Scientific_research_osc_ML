# Фаза 2.7: Высококачественные эксперименты для научных статей

## Цель

Повторение и расширение экспериментов Фазы 2.5 с методологическими улучшениями,
необходимыми для публикации в высококачественном журнале:
- 5-кратная стратифицированная перекрёстная проверка (K-Fold CV)
- Множественные seeds для оценки стабильности обучения
- Полный перебор: типы данных × стратегии выборки × модели × сложности
- Разделение экспериментов на контексты двух статей

## Структура экспериментов

### Циклы (от внешнего к внутреннему):

1. **Fold (1-5)**: Перекрёстная проверка на разных разбиениях данных
2. **Seed (0-4)**: Проверка стабильности обучения (seeds: 42, 7961, 15773, 23539, 31649)
3. **Experiment**: Тип данных × стратегия выборки (полное декартово произведение)
4. **Model**: Архитектура нейросети (зависит от контекста статьи)
5. **Complexity**: Уровень сложности модели (light/medium/heavy)

### Типы входных данных (feature_mode):
- `raw` — Сырые сигналы (8 каналов)
- `phase_polar` — Фазные полярные координаты (амплитуда + фаза)
- `phase_complex` — Фазные комплексные (Re + Im)
- `symmetric` — Симметричные составляющие (Re + Im)
- `symmetric_polar` — Симметричные составляющие (амплитуда + фаза)
- `power` — Мощность (P, Q)
- `alpha_beta` — Преобразование Кларк

### Стратегии выборки (sampling):
- `none` — Dense: все 320 точек
- `stride` — Прореживание с шагом 16 (→ 18 точек для спектральных)
- `snapshot` — Мгновенный снимок (2 точки для спектральных, 64 для raw)

### Модели:

| Модель | Статья 1 | Статья 2 | Ограничения |
|--------|----------|----------|-------------|
| SimpleMLP | ✓ | ✓ | — |
| SimpleCNN | ✓ | ✓ | Не работает на snapshot спектральных |
| ResNet1D | ✓ | ✓ | Не работает на snapshot спектральных |
| SimpleKAN | ✓ | ✓ | — |
| ConvKAN | ✓ | ✓ | Не работает на snapshot спектральных |
| PhysicsKAN | ✓ | ✗ | Только phase_polar |
| cPhysicsKAN | ✓ | ✗ | Только phase_polar |
| rPhysicsKAN | ✓ | ✗ | Только phase_polar |
| PatchTST | ✓ (TODO) | ✓ (TODO) | Фиксированная сложность |
| TimesNet | ✓ (TODO) | ✓ (TODO) | Фиксированная сложность |

### Контексты статей:
- **Статья 1 (paper1)**: Сравнение моделей. Фиксированный тип данных (phase_polar), все стратегии выборки, ВСЕ модели.
- **Статья 2 (paper2)**: Сравнение входных данных. Все типы данных, все стратегии, базовые модели (без Physics*KAN).

## Методология

### Перекрёстная проверка (5-Fold CV)
- Разбиение по файлам (осциллограммам), НЕ по отдельным точкам
- Стратификация по комбинации меток (has_ML1, has_ML2, has_ML3)
- Результат сохранён в `data/ml_datasets/cv_splits.json`
- Баланс: ~198 файлов в тесте, ~792 в обучении каждый fold
- Классы в тесте: ML1≈27-28, ML2≈10-11, ML3≈20-21 файлов

### Решение по валидации (train/valid/test)
Решено НЕ выделять отдельную валидационную выборку, т.к.:
- Малый размер выборки (особенно ML2: всего 51 файл)
- В предыдущих опытах (2.5) разбиение на 3 части давало нестабильные результаты
- Сохраняется best_model.pt по val_loss, но основная оценка — по финальной модели

### Аугментация
Всегда включена (по результатам Фазы 2.5): инверсия фаз, масштабирование, перемешивание фаз (исследования проводились, но множить здесь не стали, принимается за данность).

### Взвешивание классов
BCEWithLogitsLoss с pos_weight (взвешивание по частоте событий, небольшое исследование проводилось, но множить здесь не стали, принимается за данность).

## Запуск

### Ручной (IDE):
Настроить константы в блоке `if __name__ == "__main__"` файла `run_phase2_7_all.py`.

### CLI:
```bash
# Полный прогон Статьи 1
python scripts/phase2_experiments/run_phase2_7_all.py --article paper1 --fold all --seed_idx all --epochs 30

# Быстрый тест (1 эпоха)
python scripts/phase2_experiments/run_phase2_7_all.py --fold 1 --seed_idx 0 --epochs 1 --no-skip

# Параллельный запуск на сервере (по fold-ам)
python scripts/phase2_experiments/run_phase2_7_all.py --fold 1 --seed_idx all --article paper2 --epochs 30
python scripts/phase2_experiments/run_phase2_7_all.py --fold 2 --seed_idx all --article paper2 --epochs 30
# ...и т.д.

# Конкретная модель/эксперимент
python scripts/phase2_experiments/run_phase2_7_all.py --fold 1 --seed_idx 0 --exp phase_polar_stride --model SimpleCNN --complexity medium --epochs 30
```

### Параметры CLI:
| Параметр | Описание | По умолчанию |
|----------|----------|--------------|
| `--article` | paper1/paper2/both | both |
| `--fold` | 1-5 или all | all |
| `--seed_idx` | 0-4 или all | 0 |
| `--exp` | feature_mode_sampling или all | all |
| `--model` | имя модели или all | all |
| `--complexity` | light/medium/heavy/all | all |
| `--epochs` | число эпох | 30 |
| `--samples` | окон на файл | 12 |
| `--no-skip` | не пропускать обученные | false |

## Результаты

Сохраняются в `experiments/phase2_7/Exp_2.7_f{fold}_s{seed}_{model}_{complexity}_{feature}_{sampling}/`:
- `config.json` — параметры эксперимента
- `metrics.jsonl` — метрики по эпохам
- `best_model.pt` — лучшая модель по val_loss
- `final_model.pt` — финальная модель

## Оценка общего количества экспериментов

### Статья 1 (paper1):
- 5 folds × 5 seeds × 3 sampling × 8 моделей × 3 сложности = **1800** экспериментов
  (минус несовместимые комбинации ≈ **1500**)

### Статья 2 (paper2):
- 5 folds × 5 seeds × 7 feature_modes × 3 sampling × 5 моделей × 3 сложности = **7875** экспериментов
  (минус несовместимые ≈ **6000**)

### Полный перебор (both):
- ~8000-9000 экспериментов (с учётом пересечений)
