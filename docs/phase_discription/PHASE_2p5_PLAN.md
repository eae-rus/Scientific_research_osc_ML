## 📅 Детальный план работ

### Этап 1: Рефакторинг работы с данными (Data Engineering)
*Самый важный этап. Мусор на входе = мусор на выходе.*

1.  **Аудит и селектор сигналов:**
    *   **Проблема:** В данных "зоопарк" колонок: `IA, IB, IC, IN`, `UA BB` (шины), `UA CL` (линия), линейные `UAB` и т.д.
    *   **Задача:** Реализовать умный селектор в `OscillogramDataset`.
        *   Если есть фазные `UA`, брать их (причём они могут быть 0 все - значит нет, а могут быть шум, то есть есть. Надо подумать, как заносятся).
        *   Если есть только линейные `UAB`, использовать их (но помнить, что $U_0$ из них не получить).
        *   Если нет СШ (BB) - то пробовать работать по КЛ (CL).
        *   А если и их нет - то базисным является ток фазы А.
        *   Входной тензор модели должен быть динамическим или фиксированным максимальным (с заполнением нулями отсутствующих каналов). Разобраться с возможностью определения присутствующих сигналов.
    *   **Задача:** Реализовать логику расчета симметричных составляющих для линейных напряжений (только $U_1, U_2$).

2.  **Глобальная нормализация:**
    *   **Задача:** Написать скрипт `scripts/calculate_dataset_stats.py`.
    *   Проход по всему Train-сету и расчет `Mean` и `Std` для каждого типа канала (Токи, Напряжения).
    *   Сохранение в `stats.json`.
    *   Внедрение загрузки этих статистик в `OscillogramDataset` для Z-нормализации и проверки физической нормализации входных данных (Если они прям сильно выходят за диапазоны - значит не были нормированы от исходных значений).

3.  **Sliding Window & Random Sampling:**
    *   **Задача:** Модифицировать `Dataset`. Вместо `start_idx=0` реализовать логику:
        *   *Train:* Случайный выбор окна длиной `window_size` в пределах файла. В связи с этим решить с объёмом обучающего набора (Я раньше фиксировал количество примеров в одной эпохе и прост случайные брал осциллограммы и в них рандомные куски)
        *   *Val/Test:* Скользящее окно с шагом (например, 50% перекрытия) для покрытия всей осциллограммы.
    *   **Параметр окна:** Установить базовый размер окна **200 мс** (320 точек при 1600 Гц). Это физически обоснованное время.

### Этап 2: Стратегии формирования входа (Input Strategies)
*Здесь мы решаем проблему скорости KAN и физического смысла.*

1.  **Переход на Модуль/Угол:**
    *   **Задача:** Заменить `Re/Im` на `Magnitude/Angle`.
    *   **Важно:** Фаза (Angle) сама по себе бесполезна, так как зависит от момента начала окна.
    *   **Решение:** Считать разность фаз относительно **Напряжения Фазы А** (или линейного напряжения, или Тока А, если напряжения нет). $\phi_{rel} = \phi_{x} - \phi_{Ua}$. Это делает признак инвариантным к сдвигу во времени.

2.  **Реализация 3-х стратегий сэмплирования (для окна 200 мс / 320 точек):**
    *   **Стратегия A (Dense/Raw):** Все 320 точек сырого сигнала (или 320 рассчитанных векторов RMS/Phase). *Самая тяжелая.*
    *   **Стратегия B (Strided/Downsampled):** Расчет признаков (RMS/SymComp) с шагом (stride) **16 точек**.
        *   Итоговый размер: $320 / 16 = 20$ временных отсчетов.
        *   Это идеальный баланс для KAN (мало точек, много смысла).
    *   **Стратегия C (Snapshot/Sparse):** Взятие только двух точек: $t_{current}$ и $t_{start}$ (начало окна, -200мс).
        *   Итоговый размер: 2 временных отсчета.
        *   Проверка гипотезы: "Достаточно знать, что было 'до' и что 'сейчас', чтобы понять тип аварии".

### Этап 3: Работа с целевыми метками (Targets)

1.  **Multi-Label подход:**
    *   **Задача:** Изменить кодирование `y`. Вместо одного числа (класс 0..N) выдавать вектор бинарных флагов $[0, 1, 0, 1, ...]$.
    *   Использовать `BCEWithLogitsLoss`.

2.  **Два уровня детализации:**
    *   **Level 1 (Base):** 4 класса.
        *   `Normal` (Нет флагов)
        *   `ML_1` (Рабочая коммутация)
        *   `ML_2` (Аномалия)
        *   `ML_3` (Авария)
        *   *Примечание:* Это требует маппинга исходных `ML_...` колонок в эти 4 категории. И то есть сигнал `ML_1_1` относится к рабочей коммутации, важна только первая цифра.
    *   **Level 2 (Full):** Все уникальные флаги.

### Этап 4: Адаптация моделей

1.  **Обновление входных слоев:**
    *   Модели должны принимать динамическое количество каналов (в зависимости от файла конфигурации `input_channels`).
    *   Для KAN/MLP: Учет изменения размера входа при смене стратегии (20 точек vs 320 точек).

2.  **Обновление выходных слоев (Heads):**
    *   Реализовать `MultiHead` или просто конфигурируемый выходной слой.
    *   Head 1: 4 нейрона (для Base классов).
    *   Head 2: N нейронов (для всех флагов).
    *   *Гипотеза:* Можно учить одну сеть с двумя головами одновременно (Multi-task learning).

## ⚠️ Риски
-   **Отсутствие фазы А:** Для расчета относительного угла нужен опорный вектор. Если $U_A$ и $I_A$ отсутствуют или зашумлены, относительный угол "поплывет". *Митигация:* Использовать фазу B или C, если А недоступна, или использовать приращения ($\Delta \phi$).
-   **Несбалансированность линейных/фазных:** Модель, обученная на файлах с фазными напряжениями, может плохо работать на файлах только с линейными. *Митигация:* Обучать разные модели или использовать Dropout каналов (случайно занулять фазные напряжения в обучении, заставляя сеть учить линейные).
