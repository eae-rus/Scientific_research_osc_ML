# План Фазы 4: Развитие органов РЗА (Advanced)

-> **Статус:** Планируется  
> **Цель:** перейти от классификации окон к полноценным «органам РЗА»: временной разметке событий, устойчивой логике принятия решений и анализу аномалий.

---

## 🎯 Основные цели фазы

> **⚠️ Ключевая проблема из Фаз 2.5–2.6:** Свёрточные модели показывают лучшую точность, но страдают от «размазанности» (smearing) границ событий. Эта фаза должна решить эту проблему.

1. **Сегментация осциллограмм:** разметка по времени (начало/конец события, смена режима).
2. **Архитектуры с временным контекстом:** 1D U‑Net, TCN, KAN‑U‑Net.
3. **Аномалия и восстановление:** автоэнкодеры для поиска нетипичных режимов.
4. **Логика органа РЗА:** пост‑обработка предсказаний (гистерезис, задержки, фильтрация).
5. **Anti-Smearing:** Специализированные методы борьбы с размазанностью (Boundary Loss, CRF, Debouncing).

---

## 📦 Область экспериментов (фиксированная)

- **Данные/признаки:** использовать лучшие настройки из Фазы 3 (ожидаемо `phase_polar` + `stride`).
- **Метрики сегментации:** IoU, F1‑score по времени, точность границ (delay/error).
- **Метрики аномалий:** ROC‑AUC, PR‑AUC, пороговые уровни.

---

## 📅 Детальный план работ

### Этап 1: Подготовка данных для сегментации

1. **Схема разметки времени:**
    *   [ ] Определить временную гранулярность (шаг окна и перекрытие).
    *   [ ] Определить, как агрегируются метки в интервале окна.

2. **Dataset для сегментации:**
    *   [ ] Создать Dataset/Loader, возвращающий последовательность меток на отрезке.
    *   [ ] Описать правила «смягчения» границ (окна перехода).

3. **Базовые метрики:**
    *   [ ] Реализовать расчёт задержки срабатывания и ошибку позиционирования границ.

---

### Этап 2: Архитектуры сегментации

1. **Базовые модели:**
    *   [ ] 1D U‑Net (baseline) — классический encoder-decoder с skip-connections.
    *   [ ] TCN (Temporal Convolutional Network) — dilated convolutions для большого рецептивного поля.

2. **KAN‑варианты:**
    *   [ ] KAN‑U‑Net (замена FC/Conv‑блоков на KAN‑слои).
    *   [ ] Упрощённый KAN‑блок в бутылочном горлышке (для интерпретируемости).

3. **Борьба с размазанностью границ (Boundary-Aware Training):**
    *   [ ] **Boundary Loss:** Добавить в loss-функцию штраф за нечёткие границы. Формула: $L_{boundary} = ||\nabla y_{pred} - \nabla y_{true}||$.
    *   [ ] **Focal Loss:** Для фокусировки на трудных переходах между классами.
    *   [ ] **Transition Weighting:** Увеличить вес окон вокруг моментов смены режимов.

4. **Сравнение:**
    *   [ ] Оценка качества и скорости инференса.

---

### Этап 3: Аномалия и восстановление

1. **Autoencoder/Variational AE:**
    *   [ ] Реализовать AE/VAE для реконструкции сигналов.
    *   [ ] Выделить порог аномалии по ошибке восстановления.

2. **KAN‑AE:**
    *   [ ] Вариант с KAN‑слоем в bottleneck.

3. **Связь с органом РЗА:**
    *   [ ] Совместить аномальный сигнал и сегментацию (ensemble‑логика).

---

### Этап 4: Логика органа РЗА (Post-Processing & Sharpening)

**Проблема:** Пользователь отмечает "размазанность" (smearing) предсказаний свёрточных моделей на переходах между режимами.
**Задача:** Улучшить четкость границ событий.

1.  **Anti-aliasing / Deblurring:**
    *   [ ] **CRF (Conditional Random Fields):** Использование классического ML поверх выходов нейросети для учета вероятностей переходов (напр., из "Аварии" в "Норму" за 1 такт перейти нельзя).
    *   [ ] **Confidence Thresholding + Debouncing:** Фильтрация "дребезга" контактов/предикторов. Если класс сменился на 1 отсчет — игнорировать.

2.  **Фильтрация предсказаний (РЗА Логика):**
    *   [ ] Гистерезис, минимальная длительность события (напр., 5 мс).
    *   [ ] Правила подтверждения (k‑из‑n окон).

---

### Этап 5: Инженерные ограничения и тесты

1.  **Метрики:**
    *   [ ] **Smearing Width (Ширина размытия):** Среднее количество тактов, за которое вероятность класса меняется от 0.1 до 0.9. (Цель: минимизировать).
    *   [ ] Delay (задержка срабатывания в мс).
    *   [ ] Jitter (дрожание метки).

2.  **Контроль ложных срабатываний:**
    *   [ ] Проверка на "шумовых" файлах (Normal, Switches).

---

## ✅ Критерии завершения фазы

- [ ] Получены модели сегментации с приемлемой задержкой и точностью.
- [ ] Реализован модуль пост-обработки (Debouncing/CRF), убирающий "размазанность".
- [ ] Реализована базовая логика органа РЗА.
- [ ] Проведено сравнение U‑Net/TCN/КАН‑вариантов.


---

## 📌 Примечания

- **Зависимость от Фазы 3:** Использовать выбранную в Фазе 3 библиотеку KAN для KAN-U-Net.
- **Проблема размазанности:** Это ключевая техническая проблема Фазы 4. Приоритет отдать этапу 4 (Post-Processing).
- Если качество сегментации недостаточно, допускается упрощение до «точки срабатывания» (detector‑mode).
- PDR/РНМ остаётся отложенным до появления разметки.
