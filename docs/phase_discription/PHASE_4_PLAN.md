# План Фазы 4: Построение единой сложной модели Physical KAN-Transformer

-> **Статус:** В процессе (Этап 2 — SSL pretrain 100 эпох запущен 17.03.2026, Этап 4 — fine-tuning pipeline реализован)
**Цель:** Создать единую модель Трансформера с физическим индуктивным смещением (PINN) в лимите 8 ГБ VRAM. Основной фокус текущего этапа — **обучение без учителя (self-supervised)** на векторных спектральных данных (восстановление входных параметров). Предсказание классов (fine-tuning) с анти-smearing логикой будет реализовано позже, где под зонами понимаются окна по 16 точек (половина периода), или даже меньше.

---

## 🎯 Основные концепции и векторы развития

1. **Единая Seq2Seq модель:** окно 320 точек сохраняется, но надо это понимать не как количество точек, а как 10 периодов промышленной частоты, потому что потом буду датасеты с другой дискретизацией. Предсказание формируется по всей последовательности (по сути по зонам stride 16), а не только по последней точке, как было ранее.
2. **Attention:** Используется стандартный механизм Attention на всех признаках без принудительного разделения матриц $Q$ и $K$ на токи и напряжения (модель должна выучить зависимости самостоятельно, если это потребуется).
3. **Сохранение физблоков:** операции умножения/деления остаются в stem-части как обязательный inductive bias. Стоит отметить, что исходно стоит использовать модель основанную на комплексных числах (cPhysicsKAN), а в будущем она будет улучшена до "rPhysicsKAN" у которой угол может ещё уменьшать и модуль. И надо не забывать про нормализацию подаваемых сигналов
4. **Обучение без учителя (Self-supervised):** Первичное внедрение и отладка пайплайна происходят на **текущем датасете**. Расширение на огромный внешний неразмеченный датасет вынесено в будущие задачи (потребует адаптации под разные частоты дискретизации, где окно всегда = 10 периодам, а stride = 0.5 периода).
5. **KAN в «узких местах»:** KAN используется точечно (полносвязные Dense-слои, не свёртки). Тоесть там, где это будет вычислительно оправдано на твой взгляд и иметь достатчоный смысл.
   *Задача агенту: Разобраться и протестировать архитектурный порядок в Stem-блоке — должна ли KAN-нелинейность применяться ДО или ПОСЛЕ физических операций умножения/деления. Проанализировать, нужно ли делать связи "все со всеми" на этом этапе или оставить расчеты изолированными в рамках одного временного шага.*

### Этап 4.5: Fine-tuning на симулированном датасете `Simulated_OZZ_v1`

> **Цель:** Проверить способность Physical KAN-Transformer распознавать **типы ОЗЗ/ДПОЗЗ** (устойчивое vs три теории перемежающейся дуги + класс «нет ОЗЗ») на полностью размеченном RTDS-датасете. Предобучения (SSL) не требуется — стартуем с random init на лёгкой модели.

**Особенности датасета** (см. [docs/Description_OZZ_dataset.md](../Description_OZZ_dataset.md)):
- Путь: `data/Simulated_OZZ_v1/*.csv` (~19 200 файлов по 0.5 с, суммарно ~62 ГБ)
- Fs ≈ 38 461.5 Гц (определяется из колонки `Time`, dt = 2.6e-5 с), **без ресэмплирования** — FFT считается напрямую по сырым данным (samples_per_period ≈ 769).
- Шаблон имени: `OZZ_X_R_L_P_T.csv` или `OZZ_X_R_L_P_I_T.csv` (X=тип дуги 1–4, R=Ом, L=% длины, P=фаза 1–3, I=момент зажигания, T=мс).
- Сигналы: `S1_VA/VB/VC` → `UA/UB/UC`; `IA/IB/IC`; `3I0` → `IN`; `3U0` → `UN` (каналы нулевой последовательности — **не нули**, а реальные измерения!).
- **Ground truth разметка per-timestep** в самих CSV: флаги `Stable`, `Petersen`, `Peters_Slepian`, `Beliakov2` = 1 только в моменты горения дуги; в одном файле активна ровно одна теория.

**Целевые классы** (4 + implicit Normal):
- `Target_OZZ_Stable` — устойчивое однократное пробое (ОЗЗ).
- `Target_OZZ_Petersen` — перемежающаяся дуга по Петерсену (ДПОЗЗ).
- `Target_OZZ_PetersSlepian` — ДПОЗЗ по Петерсу–Слепяну.
- `Target_OZZ_Beliakov` — ДПОЗЗ по Белякову.
- Класс «Normal» — все четыре равны 0 (отсутствие ОЗЗ).

**Архитектура загрузки данных** (62 ГБ ≫ 32 ГБ RAM):
- Lazy-загрузчик `SimOZZLazyDataset`: файлы грузятся по требованию, кэшируются LRU (≤ 500 файлов ≈ 500 МБ).
- Индексация: `SimOZZFileIndex.from_directory()` — считывает только число строк и dt для каждого файла (без загрузки данных).
- Каждый `__getitem__`: загружает CSV → извлекает окно → аугментация → FFT → (X, Y).

**Параметры спектральной обработки:**
- Stride = 1/8 периода ≈ 96 отсчётов (вместо 1/2 в старых экспериментах). В будущем возможно 1/16, 1/32.
- Окно FFT = 1 период (769 отсчётов), окно данных = 10 периодов (7 690 отсчётов).
- Гармоники: 9 стандартных + 4 суб-периода [2, 4, 6, 10] → 220 каналов (с симм. составл.).
