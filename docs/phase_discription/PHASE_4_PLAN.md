# План Фазы 4: Построение единой сложной модели Physical KAN-Transformer

-> **Статус:** В планировании
> **Цель:** создать единую модель Трансформера с физическим индуктивным смещением (PINN), которая обучается в 2 этапа (self-supervised + fine-tuning), работает в лимите 8 ГБ VRAM и уменьшает smearing за счёт предсказания зон/последовательности, а не одной точки.

---

## 🎯 Основные концепции и векторы развития

1. **Единая Seq2Seq модель:** окно 320 точек сохраняется, предсказание формируется по всей последовательности (или по зонам stride 16/32), а не только по последней точке.
2. **Physical Attention:** $Q$ из токов, $K$ из напряжений, проверка гипотезы, что $Q \cdot K^T$ стабильно кодирует физическое взаимодействие и помогает границам событий.
3. **Сохранение физблоков:** операции умножения/деления остаются в stem-части как обязательный inductive bias.
4. **Обучение без учителя на большом неразмеченном датасете:** сначала восстановление входных параметров, затем supervised дообучение на целевых метках.
5. **KAN в «узких местах»:** KAN используется точечно (stem/FFN/heads) с учётом лимита 8 ГБ VRAM.

### Этап 3: Архитектура Physical KAN-Transformer

1. **Stem / Patch embedding (PhysicsKAN-идея):**
   - [ ] Сформировать stem-блок с физоперациями (умножение/деление) до основного энкодера.
   - [ ] Проверить два варианта: stem только на физпризнаках и stem на `raw + physics`.

2. **Physical Attention:**
   - [ ] Реализовать attention-слой с разделением ток/напряжение (`Q`/`K`).
   - [ ] Добавить позиционное кодирование (RoPE и/или синусоидальное) и сравнить.
   - [ ] Встроить маскирование отсутствующих каналов в attention score.

3. **KAN-блоки внутри Transformer FFN:**
   - [ ] Реализовать адаптер backends для FFN по аналогии с текущим `kan_backend` подходом (минимум: `baseline`, `efficient`, `fast`, `cheby`).
   - [ ] Добавить в скрининг **`KAT (Rational KAN)`** (`Adamdad/kat`) — рациональные функции $P(x)/Q(x)$. Особенно перспективен, так как физическая идея проекта строится на делении ($Z=U/I$); слой, аппроксимирующий зависимости через отношение полиномов, может дать прирост при классификации КЗ и переходных процессов.
   - [ ] ❌ **Wav-KAN / torch-wavelet-kan исключить из пайплайна** при использовании FFT-признаков: данные уже находятся в частотной области (гармоники), применение вейвлет-преобразования к спектру Фурье математически некорректно и запутает модель. Вейвлеты применимы только к сырым временным рядам.
   - [ ] Итоговый приоритет backend-ов: `fast-kan` (скорость) → `cheby` (стабильность без grid) → `KAT` (физическая интерпретируемость) → `efficient-kan` только в stem.
   - [ ] Провести ограниченный скрининг по VRAM/скорости/стабильности.
   - [ ] Зафиксировать минимально жизнеспособный backend для фазы 4 (один основной + один резервный).

4. **Исследование дополнительной KAN-модернизации:**
   - [ ] Проверить, есть ли практический смысл KAN-замены вне FFN (например, выходные головы), не выходя за лимиты памяти.

## 📌 Примечания

- **Главное ограничение:** VRAM 8 ГБ, поэтому решения оцениваются не только по точности, но и по вычислительной стоимости.
- **Ключевой принцип:** не заменять всё на KAN, а точечно усиливать узлы, где это даёт максимальный прирост при контролируемой памяти.
- **Главный фокус:** физически интерпретируемая архитектура (Physics stem + Physical Attention) и уменьшение smearing на границах событий.
