## 🎯 Общее описание проекта

### Научная цель
Исследование применимости Kolmogorov-Arnold Networks (KAN) для задач релейной защиты и автоматики (РЗА) в сравнении с традиционными архитектурами глубокого обучения (CNN, MLP). Основной фокус на интерпретируемости, эффективности и способности работать с комплексными сигналами.

### Ключевые задачи
1. **PDR (Power Direction Relay)** — Определение направления мощности (ранее РНМ).
2. **Классификация аварийных режимов** — Распознавание типов повреждений (3-4 основных типа, с перспективой расширения).
3. **Сегментация осциллограмм** — Разметка событий во времени (не просто классификация файла, а определение границ режимов).
4. **Encoder-Decoder архитектуры** — Восстановление сигналов, детекция аномалий и генерация признаков.
5. **Сравнительный анализ** — KAN vs CNN vs MLP.

### Ограничения
- Вычислительная эффективность (модели до ~10^6 параметров)
- Работа на промышленных процессорах (не GPU)
- Физическая обоснованность операций

## 🛠️ Фаза 1: Подготовка инфраструктуры

### Задача 1.1: Рефакторинг ML модуля

**Цель:** Создать гибкую структуру для поддержки разных архитектур (не только KAN).

**Действия:**
```
osc_tools/ml/
├── __init__.py
├── models/
│   ├── __init__.py
│   ├── base.py              # Базовые классы
│   ├── mlp.py               # Классические полносвязные сети (Baseline)
│   ├── cnn.py               # 1D CNN (Baseline)
│   ├── kan.py               # KAN архитектуры
│   ├── hybrid.py            # Гибридные модели (CNN+KAN, etc.)
│   └── autoencoders.py      # Encoder-Decoder модели
├── layers/
│   ├── __init__.py
│   ├── kan_layers.py        # Обертки над KAN библиотеками
│   ├── complex_ops.py       # Комплексные операции
│   └── blocks.py            # Строительные блоки (ResBlock, PDRBlock)
└── utils.py
```

### Задача 1.2: Реализация Baseline моделей

Для корректного сравнения нужны "типичные" представители:
1.  **MLP (Multi-Layer Perceptron):** Простая полносвязная сеть.
2.  **1D CNN (Convolutional Neural Network):** Стандарт де-факто для временных рядов (ResNet-like или простые свертки).

### Задача 1.3: Подготовка данных для новых задач

**Расширение форматов:**
1.  **Classification:** Один лейбл на файл (текущий подход).
2.  **Segmentation:** Лейбл для каждого отсчета времени (или окна). Требует изменения `Dataset` и `DataLoader`.
3.  **Reconstruction:** Вход = Выход (для Autoencoders).

## 🧪 Фаза 2: Базовые эксперименты и сравнение

### Задача 2.1: Сравнительный анализ на PDR

**Цель:** Показать, дает ли KAN преимущество перед MLP/CNN в задаче определения направления мощности.

**Участники:**
1.  `PDR_MLP` (Baseline)
2.  `PDR_CNN` (1D Conv)
3.  `PDR_KAN` (Kolmogorov-Arnold)

**Метрики:**
- Точность (Accuracy/F1)
- Количество параметров
- FLOPs (вычислительная сложность)
- Интерпретируемость (возможность извлечь формулу)

### Задача 2.2: Сравнительный анализ на классификации КЗ

**Задача:** Распознавание 3-4 основных типов аварий.

**Эксперимент:**
- Обучение трех типов моделей на одних данных.
- Анализ confusion matrix.
- Проверка устойчивости к шуму.

## � Фаза 2.5: Углубленная оптимизация данных и архитектур (Data-Centric Refinement)

### Задача 2.5.1: Рефакторинг работы с данными (Data Engineering)

**Цель:** Переход от статичного сэмплирования к динамическому (Sliding Window, Random Sampling) и улучшение физической интерпретируемости признаков.

**Ключевые изменения:**
- **Sliding Window & Random Sampling:** Вместо фиксированного окна с начала файла, случайный выбор окна по всей длине осциллограммы.
- **Smart Channel Selection:** Автоматический выбор доступных сигналов (фазные напряжения, линейные, токи).
- **Global Normalization:** Расчет статистики по всему датасету для Z-нормализации.
- **Input Strategies:** Сравнение Dense (все точки), Strided (downsampling), Snapshot (2 точки).
- **Аугментация:** Добавление аугментации данных.

### Задача 2.5.2: Переход к Multi-Label классификации

**Цель:** Обучение на более детализированных метках (несколько флагов одновременно).

**Уровни детализации:**
- **Base Level:** 4 класса (Normal, Рабочая коммутация, Аномалия, Авария).
- **Full Level:** Все уникальные флаги.

**Архитектуры:** Multi-Head модели с BCEWithLogitsLoss.

### Задача 2.5.3: Физически интерпретируемые признаки

**Цель:** Замена Re/Im на Magnitude/Angle относительно опорной фазы для лучшей интерпретируемости.

**Эксперименты:**
- Mag/Angle vs Re/Im.
- Relative angles (относительно фазы A).
- Сравнение стратегий сэмплирования.

## ⚡ Фаза 4: Развитие органов РЗА (Advanced)

### Задача 4.1: Сегментация осциллограмм (Full Waveform Labeling)

**Задача:** Вместо одного предсказания на файл, модель должна выдавать класс события для каждого момента времени (или окна).

**Архитектуры:**
- **U-Net (1D):** Классический энкодер-декодер с skip-connections.
- **KAN-U-Net:** Замена сверток или полносвязных слоев на KAN-слои.
- **TCN (Temporal Convolutional Network):** Свертки с дилатацией.

**Цель:** Точное определение времени начала и конца аварийного режима.

### Задача 4.2: Encoder-Decoder и Anomaly Detection

**Задача:** Обучение представлениям (representation learning) и поиск аномалий.

**Архитектуры:**
- **Autoencoder (AE):** Сжатие и восстановление сигнала.
- **Variational Autoencoder (VAE):** Генеративная модель.
- **KAN-AE:** Использование KAN в узком горлышке (bottleneck) для получения интерпретируемого латентного пространства.

**Применение:**
- Детекция аномалий (высокая ошибка восстановления = аномалия).
- Pre-training для других задач.
